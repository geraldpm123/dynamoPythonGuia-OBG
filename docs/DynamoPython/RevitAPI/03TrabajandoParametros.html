3.2. Unidades de Revit​

Revit a nivel de usuari opermite trabajr con muchos tipos de unidads de acuerdo a la necesidad del usuario, por l oque tenemos una amplia serie de Unidades, pero internamente como sabemos los valores almacenados on de tipo double por lo que, es necesario saber en que unidades se gurdan estos valorese en Revit.

Revit internamente trabaja con las siguientes unidades:

Unidad BaseUnidad en Revit
LongitudPie (ft\mathrm{ft}ft)
AnguloRadian
MasaKilogramo (kg\mathrm{kg}kg)
TiempoSegundo (s\mathrm{s}s)
Intensidad de corrienteAmperio (A\mathrm{A}A)
TemperaturaKelvin (K\mathrm{K}K)
Intensidad LumisosaCandela (cd\mathrm{cd}cd)

Estas son las unidades basicas, y para las derivadas seria una combinacion de estas, por ejemplo las unidades de Fuerza seria: kg⋅ft/s2\mathrm{kg \cdot ft/s^2}kg⋅ft/s2

3.3. Transacciones​

Dynamo proporciona su propio marco de transacciones para trabajar con RevitAPI. Esto significa que el script de Python se ejecutará en el contexto de una transacción general de Dynamo.

Si está escribiendo código RevitAPI que requiere una transacción, puede utilizar el archivo .TransactionManager

TransactionManager.EnsureInTransaction(): Inicializa la transacción de Dynamo TransactionManager.TransactionTaskDone(): Le dice a Dynamo que hemos terminado de usar la transacción TransactionManager.ForceCloseTransaction(): Indica a Dynamo que confirme la transacción activa. Esto es más lento que , por lo que solo úselo cuando realmente necesite cerrar la transacción para que su script funcione.TransactionTaskDone() Para la secuencia de comandos, en cualquier lugar en el que normalmente crearía una nueva transacción de RevitAPI y, a continuación, llamaría a , utilizará en su lugar . En cualquier lugar al que normalmente llamaría , en su lugar usará . En cualquier lugar en el que realmente necesite que finalice la transacción (situaciones en las que desee operar en el modelo después de que se produzca la regeneración), puede usar .Transaction.Start()TransactionManager.EnsureInTransaction()Transaction.Commit()TransactionManager.TransactionTaskDone()TransactionManager.ForceCloseTransaction()

import clr

# Import DocumentManager and TransactionManager
clr.AddReference("RevitServices")
import RevitServices
from RevitServices.Persistence import DocumentManager
from RevitServices.Transactions import TransactionManager

# Get the document
doc = DocumentManager.Instance.CurrentDBDocument

# "Start" the transaction
TransactionManager.Instance.EnsureInTransaction(doc)

# Create a reference point (requires a transaction)
refPt = doc.FamilyCreate.NewReferencePoint(XYZ(0, 0, 0))

# "End" the transaction
TransactionManager.Instance.TransactionTaskDone()

También puede utilizar las subtransacciones de RevitAPI junto con (pero no como sustituto) del archivo . Las subtransacciones le darán la posibilidad de revertir los cambios. No hay forma de revertir la transacción principal de Dynamo.TransactionManager

3.4. Modificación de parámetros​

import clr

clr.AddReference('RevitAPI')
from Autodesk.Revit.DB import *

clr.AddReference("RevitServices")
from RevitServices.Persistence import DocumentManager
from RevitServices.Transactions import TransactionManager

doc = DocumentManager.Instance.CurrentDBDocument

# Inputs
muro = UnwrapElement(IN[0])

# Obtenemos el parámetro Comentarios
paramComments = muro.get_Parameter(BuiltInParameter.ALL_MODEL_INSTANCE_COMMENTS)

# Inicializar la transaction
TransactionManager.Instance.EnsureInTransaction(doc)
# Asignarle un valor a este parámetro
resultado = paramComments.Set("Hola Mundo")
# Finalizar la transacción
TransactionManager.Instance.TransactionTaskDone()

# Asigne la salida a la variable OUT.
OUT = resultado
import clr

clr.AddReference('RevitAPI')
from Autodesk.Revit.DB import *

clr.AddReference("RevitServices")
from RevitServices.Persistence import DocumentManager
from RevitServices.Transactions import TransactionManager

doc = DocumentManager.Instance.CurrentDBDocument
# Entrada de datos
muro = UnwrapElement(IN[0])
nivel = UnwrapElement(IN[1])
elementId_nivel = nivel.Id

# Obtenemos el parámetro Restricción superior
paramResSup = muro.get_Parameter(BuiltInParameter.WALL_HEIGHT_TYPE)

# Inicializar la transacción
TransactionManager.Instance.EnsureInTransaction(doc)
# Asignarle un valor a este parámetro
resultado = paramResSup.Set(elementId_nivel)
# Finalizar la transacción
TransactionManager.Instance.TransactionTaskDone()

# Asigne la salida a la variable OUT.
OUT = resultado

En PyRevit tambien podriamos hacer el mismo ejemplo, solo que no tendremos disponibles las variables IN y OUT, pero podemos hacer lo siguiente:

Ejemplo: Modificar parametro con PyRevit
# -*- coding: utf-8 -*-
from Autodesk.Revit.DB import *

# Variables comunmente necesarias (ya estan cargadas en PyRevit)
uiapp = __revit__
app = __revit__.Application 
uidoc = __revit__.ActiveUIDocument
doc = __revit__.ActiveUIDocument.Document

# Obtenemos el muro y nivel a partir del valor de sus Ids
muro = doc.GetElement(ElementId(381061))
nivel = doc.GetElement(ElementId(381018))

# Obtenemos el id del nivel
elementId_nivel = nivel.Id

# Obtenemos el parametro Restriccion superior
paramResSup = muro.get_Parameter(BuiltInParameter.WALL_HEIGHT_TYPE)

# Realizamos la modificacion del parametro dentro de un transaccion
trans = Transaction(doc, "Modificación Mia")
# Inicializar la transaccion
trans.Start()
# Asiganrle un valor a este parametro
resultado = paramResSup.Set(elementId_nivel)
# Finalizar la transaccion
trans.Commit()
